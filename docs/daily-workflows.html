AI Automation Starter Kit

Turn daily source files into reviewed, approved drafts

Choose the work, date, and workspace. Codex asks for missing decisions, creates a source-grounded draft, and leaves final review and approval to a person.

Project overview · Start with Codex · 日本語ガイド

Product boundary: local file intake, questions, drafts, PIN approval, audit, and next-period setup. It does not automatically send messages, make payments, post ledgers, correct attendance, make HR decisions, or commit to customers.

The seven-step loop

1 ChooseSelect one of ten jobs and enter the first date.
2 CreateEnter a name, approver, and PIN. The folders are created.
3 Add filesAdd approved examples, standing references, and today's inputs.
4 InspectCheck accepted, rejected, and skipped files.
5 AnswerAnswer policy and priority questions AI cannot decide.
6 DraftGenerate and review the sourced draft and uncertainties.
7 ApproveApprove locally, enter the next date, and repeat.

Start with Codex

No OpenAI API key is required. This local drafting flow uses a signed-in ChatGPT and Codex setup, with Codex CLI as the local runner. Ask Codex: “Set up this project's daily workflow automation for a beginner.” Or start the UI directly:

ai-automation-kit office-workspace packs
ai-automation-kit office-workspace serve --root "$HOME/AI-Office-Workspaces"

Daily packs use YYYY-MM-DD; the monthly report uses YYYY-MM.

Folder map

  • 01_APPROVED_PAST_OUTPUTS: approved examples for structure and tone.
  • 02_PAST_SUPPORTING_FILES: recurring policies and definitions.
  • 03_CURRENT_INPUTS/date: today's source files.
  • 04_QUESTIONS: unresolved questions.
  • 05_DRAFTS: unapproved drafts.
  • 06_APPROVED_OUTPUTS: locally approved final copies.
  • 07_AUDIT: approval history.
Past examples are references, not model training or a correctness guarantee. When preparing the next day, reuse of the latest approved output is selected by default. Only the hash-verified approved file is passed into the next Codex input. Compare numbers, owners, dates, and claims with source files.

Approval checklist

  • No unreadable source remains unexplained.
  • Numbers, owners, and dates match sources.
  • Uncertainty is not presented as fact.
  • Unnecessary personal data is removed.
  • The responsible person reviews before external use.
This tool reduces repetitive review and drafting time. It does not replace accountable human judgment.
