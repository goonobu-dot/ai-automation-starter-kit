Daily Workflow Automation: Complete Beginner Manual

Create one local workspace, place today's files in the prepared folder, let Codex ask for missing decisions, review the generated draft, and reuse the same UI tomorrow. This manual covers installation, all ten daily packs, approval, rollover, troubleshooting, and client delivery.

Read this first: this project creates local drafts and review records. It does not automatically send messages, publish content, make payments, post accounting entries, change attendance records, make HR decisions, or commit to customers.

1. What you can accomplish

Build onceCreate a dedicated workspace for one recurring job instead of rebuilding the automation every day.
Use real examplesProvide approved past work, standing reference files, and today's inputs so the draft follows known structure and current facts.
Keep human controlCodex drafts. A responsible person reviews and approves the local final copy with a PIN.

This is useful when the repeated work has a recognizable input, a recognizable output, and a person who can approve the result. It is not a promise of fully autonomous operation.

2. The complete seven-step loop

1. ChooseSelect one of ten workflow packs and enter the first date.
2. CreateEnter a workspace name, approver, and local approval PIN.
3. Add filesAdd approved examples, recurring references, and today's materials.
4. InspectReview accepted, rejected, and skipped files before generation.
5. AnswerAnswer policy, priority, audience, and exception questions.
6. DraftGenerate a source-grounded draft and review every important fact.
7. ApproveApprove locally, prepare the suggested next date, and repeat.

3. Install and confirm Codex

No separate OpenAI API key is required. This local flow uses an existing signed-in ChatGPT and Codex environment. Codex CLI is the local runner. A ChatGPT/Codex plan, login, and supported local environment may still be required.

What to prepare

  • A macOS or Linux computer for the hardened local workspace operations.
  • Python 3.9 or newer.
  • Git.
  • Codex CLI installed and signed in.
  • A local folder where client or company workspaces may be stored safely.

Install the project

git clone https://github.com/goonobu-dot/ai-automation-starter-kit.git
cd ai-automation-starter-kit
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -e .

Confirm the installation

ai-automation-kit --version
ai-automation-kit office-workspace packs
codex login status

You should see the monthly report pack followed by ten daily packs. If Codex reports that login is required, complete the Codex login before opening the operator UI.

5. Open and use the operator UI

ai-automation-kit office-workspace serve \
  --root "$HOME/AI-Office-Workspaces"

The command prints a local address such as http://127.0.0.1:8080/. Open that address on the same computer. The server is designed for loopback access, not public internet hosting.

Create the workspace

  1. Select the workflow pack.
  2. Enter the first date in YYYY-MM-DD format, for example 2026-07-12.
  3. Enter a clear workspace name, such as Client A Daily Inquiries.
  4. Enter the person responsible for final approval.
  5. Create a local approval PIN. Do not reuse a banking, device, or company password.
  6. Select the configured local save location and create the workspace.

Add the files

01_APPROVED_PAST_OUTPUTSPast final work that a responsible person already approved. Use good examples only.
02_PAST_SUPPORTING_FILESRecurring policies, FAQs, definitions, templates, and standing instructions.
03_CURRENT_INPUTS/dateOnly the files for the current day or period.
04_QUESTIONSQuestions created when required information is missing.
05_DRAFTSGenerated drafts that have not been approved.
06_APPROVED_OUTPUTSFinal local copies approved through the UI.
07_AUDITLocal approval history and integrity records.

Inspect before generating

Select Inspect materials. Confirm the number of accepted, rejected, and skipped files. Do not continue merely because some files were accepted. Make sure the files you expected are present and readable.

Answer clarification questions

The pack asks questions such as the priority rule, materiality threshold, response window, intended audience, or escalation policy. These answers are part of the current run. If the correct answer is unknown, stop and ask the responsible person.

Generate and review

Select Generate draft. Codex runs locally against a read-only snapshot of the approved inputs. Review the draft in 05_DRAFTS. Check names, numbers, dates, owners, deadlines, and every statement that could affect a customer, employee, payment, or executive decision.

Approve locally

Enter the configured approver name and PIN. Approval copies the reviewed draft into 06_APPROVED_OUTPUTS and records its hash in the local audit chain. It does not send the document anywhere.

6. Reuse the same system tomorrow

After approval, the UI suggests the next calendar date. Reuse of the latest approved output is selected by default. Confirm the date and the displayed approved example before preparing the next period.

  1. Review the suggested next date.
  2. Confirm the displayed approved output is the example you want to reuse.
  3. Select Prepare next period.
  4. Place only the new day's source files in the new dated input folder.
  5. Inspect, answer, generate, review, and approve again.
This is reference-based reuse, not model training. The system does not permanently train a new AI model on your files. It passes the approved example into the next isolated Codex input so Codex can reference its structure and tone.

How trusted reuse is protected

  • The example must be a previously recorded approved output.
  • The approval audit identity must match.
  • The file hash is checked again at run time.
  • A changed or substituted file is rejected.
  • The trusted example is copied into the isolated input snapshot and recorded with its source path and hash.

7. What Codex does and what a person must do

Codex may

  • Read supported local files.
  • Organize and summarize source facts.
  • Identify missing information.
  • Ask clarification questions.
  • Create a local draft.
  • Reference an approved prior example.

A person must

  • Choose valid source files.
  • Answer policy and business questions.
  • Verify facts, numbers, names, and dates.
  • Approve the final local copy.
  • Decide whether and how to send or publish it.
  • Own financial, HR, legal, and customer decisions.

8. Troubleshooting

Codex is not ready or not signed in

Run codex login status. Complete the Codex installation or login, then refresh the UI. A separate API key should not be pasted into the project or browser form.

The UI does not open

Confirm the terminal process is still running and open the exact 127.0.0.1 address it printed. Try another port if the selected port is already in use.

A file was rejected or skipped

Check the supported extensions: .txt, .md, .csv, .json, .docx, .xlsx, and .pdf. Confirm the file is not a link, corrupted file, oversized file, or unsupported format.

The draft is missing important information

Return to the input folder, add the missing source, inspect again, and answer any new questions. Do not invent facts in the approval step.

The next date is wrong

Correct the suggested date before preparing the next period. The system requires dates in strict YYYY-MM-DD format and rejects impossible dates.

The prior approved example is not shown

Confirm the current period is approved and an approved output exists. Only an output recorded by this workspace can be selected as a trusted next-period reference.

The approved example was edited later

The next run rejects it because the stored hash no longer matches. Restore the exact approved file or approve a new valid output through the normal workflow.

The process stopped during generation

Refresh the workspace status. Use the cancel/retry controls when available. Do not delete lock, run, or audit files manually unless the recovery documentation specifically instructs you to do so.

9. Use it for a small automation side business

Do not begin by promising complete automation. Sell a bounded drafting and review workflow whose inputs, outputs, approver, and safety boundary are visible.

A practical first-client route

  1. Ask which repeated document or review task consumes time every day.
  2. Request sanitized sample inputs and one or more approved outputs.
  3. Choose the closest pack and document what remains human-only.
  4. Build a local demonstration with synthetic or approved test data.
  5. Measure drafting time, correction count, missing information, and approval time.
  6. Run a small paid proof of concept before proposing connectors or cloud operation.
  7. Provide the workspace, this manual, recovery steps, and a clear support boundary.
Do not claim: guaranteed accuracy, autonomous customer communication, automatic legal compliance, automatic accounting approval, automatic HR judgment, or zero human work. A trustworthy offer explains both the saved work and the remaining responsibility.

10. Final approval checklist

  • Every expected source file was accepted or its absence was explained.
  • Numbers, owners, dates, deadlines, and customer statements match the sources.
  • Uncertainty is labeled rather than presented as fact.
  • Unnecessary personal or confidential information was removed.
  • The correct responsible person reviewed the draft.
  • The approved output remains local until a person decides how to use it.
  • The next period reuses only the intended approved example.
