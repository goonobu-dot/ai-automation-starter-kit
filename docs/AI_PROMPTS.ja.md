# AI用プロンプト集（生成物をClaude/ChatGPTで仕上げる）

このキットが生成する提案書や報告書は「ひな形」です。お客様の社名や数字を入れて仕上げる作業は、Claude や ChatGPT などのAIチャット（文章で指示すると文章を作ってくれるAIサービス）に任せると数分で終わります。このページは、そのまま貼り付けて使えるプロンプト（AIへの指示文）集です。

## 使い方

1. 使いたいプロンプトをまるごとコピーして、AIチャットに貼り付けます。
2. `[会社名]` のような **[ ] の置換欄**を、実際の内容に書き換えます。
3. `----- ここから貼り付け -----` の下に、キットが生成したファイルの中身（例: `.tmp/beginner-sales/proposal_one_pager.md`）を貼り付けます。

> **安全上の絶対ルール**: AIチャットには、お客様の**実データ・個人情報・APIキー・パスワードを絶対に貼らない**でください。社名と業種、件数などの数字だけで十分に仕上がります。判断に迷う情報は「◯◯社」「A業務」と伏せ字にして貼ってください。

各プロンプトの生成元ファイルの作り方は [使い方マニュアル](USER_MANUAL.ja.md)、商談全体の流れは [中小企業への自動化提案チュートリアル](TUTORIAL_SME_PROPOSAL.ja.md) を見てください。

## 1. 提案書を先方社名入りに仕上げる

`beginner-sales` の `proposal_one_pager.md` や `offer-pack` の `proposal.md` を、送れる状態に仕上げます。

```text
あなたは中小企業向けの業務自動化を提案するコンサルタントです。
以下のひな形の提案書を、次の条件で「そのまま送れる完成版」に書き換えてください。

- 宛先: [会社名]（業種: [業種]、従業員規模: [人数]名程度）
- 対象業務: [ヒアリングで聞いた業務。例: 毎週の請求書の提出催促]
- ヒアリングで聞いた数字: 月あたり[件数]件、1件あたり約[分数]分
- 提案する費用: 初回お試し導入（PoC）[金額]万円（税別）、期間[日数]日間
- 差出人: [自分の名前]

条件:
- 「＿＿」の空欄をすべて上の情報で埋めてください。
- ですます調で、専門用語（PoC、dry-runなど）には一言説明を添えてください。
- 「収益や削減額を保証する」と読める表現は使わず、「見込み」「目安」としてください。
- 実際の送信やシステム変更は行わず、下書き生成と人間の承認までである点を必ず残してください。
- A4用紙1枚に収まる分量にしてください。

----- ここから貼り付け -----
[proposal_one_pager.md の中身をここに貼る]
```

## 2. ヒアリング準備（業界別の課題仮説出し）

訪問前に、その業界でありがちな「繰り返し事務作業」の仮説を持っておくと、ヒアリング（お客様への聞き取り）の質が上がります。

```text
あなたは中小企業の業務改善に詳しいコンサルタントです。
これから [業種。例: 地域の会計事務所] の [会社名] に、業務自動化のヒアリングに行きます。

次の3つを日本語で出してください。

1. この業種で「毎週・毎月、人が手作業で繰り返している事務作業」の候補を10個。
   それぞれ、誰が・何を使って（メール/Excel/紙/電話）・どのくらいの頻度でやっていそうかの仮説付きで。
2. 上の10個のうち、「下書きの自動生成と確認リスト化」が特に効きそうなものを3つ選び、理由を一言ずつ。
3. その3つについて、ヒアリングで確認すべき追加質問を各2つ。

条件: 相手はITに詳しくない経営者です。仮説はカタカナ語を避けて、現場の言葉で書いてください。
```

## 3. デモ説明の台本化

`complete-workspace` で作ったデモ（`demo_site/index.html` と `before_after_demo.html`）を見せるときの台本を作ります。

```text
あなたは営業トークの専門家です。
中小企業の経営者（ITに詳しくない）に、業務自動化のデモ画面を見せながら説明する5分間の台本を作ってください。

前提:
- 見せる相手: [会社名] の [役職。例: 社長]
- 対象業務: [業務内容。例: 未提出の請求書を見つけて催促メールを書く作業]
- 現状: 月[件数]件、1件あたり約[分数]分かかっている
- デモの内容: 業務の流れ図、自動生成された催促メールの下書き、確認リスト（承認待ち一覧）
- 大事な前提: このデモはサンプルデータで作った見本で、メールを勝手に送ることは絶対にない

台本の構成:
1. 現状の確認（30秒）: 相手の「あるある」に共感する入り
2. デモ（3分): 画面のどこを指差して何と言うか
3. 効果の見込み（1分): 月[件数]件×[分数]分から削減時間を計算して見せる
4. 次の一歩（30秒): [金額]万円・[日数]日間の小さなお試し導入の打診

条件: ですます調。専門用語には一言説明。「保証」ではなく「見込み」と言う。想定される質問と切り返しも3つ付けてください。
```

## 4. 見積もりの根拠説明文

「なぜこの金額なのか」を聞かれたときに、そのまま話せる説明文を作ります。

```text
あなたは見積もりの説明が上手なフリーランスの先輩です。
中小企業のお客様に、業務自動化のお試し導入（PoC）の金額の根拠を説明する文章を作ってください。

前提の数字:
- 対象業務: [業務内容]
- 月あたり[件数]件 × 1件[分数]分 = 月[時間]時間の作業
- 時給換算 [金額。例: 3,000]円とすると、月あたり約[金額]円分の作業
- 提示するPoC価格: [金額]万円（税別）（目安: 月あたりの価値の1〜2か月分）
- 提示する月次運用サポート: 月額[金額]万円（税別）（目安: 月あたりの価値の3割以下）

作るもの:
1. 口頭で30秒で話せる説明（「御社の場合〜」から始める）
2. メールに書ける3〜4行の説明文
3. 「高くない？」と言われたときの切り返し(値引きではなく、範囲を小さくする方向で)

条件: ですます調。「削減額を保証する」と読める表現は使わない。数字の計算過程を見せて誠実に説明する。
```

## 5. 納品報告書の文章化

`client-report` が生成する `client_report.md` は英語見出しのひな形です。報告会でそのまま使える日本語の報告書に仕上げます。

```text
あなたは業務報告書の作成が得意なコンサルタントです。
以下の自動生成された実行レポート（英語混じり）を、お客様向けの日本語の「お試し導入 結果報告書」に書き換えてください。

前提:
- 宛先: [会社名] [承認者名]様
- 対象業務: [業務内容]
- 実施期間: [開始日]〜[終了日]
- 実測した数字: 処理[件数]件、下書き生成[件数]件、人間の確認・承認[件数]件
- 現状との比較: これまで1件[分数]分 → 確認だけなら1件約[分数]分

報告書の構成:
1. 実施内容の要約（3行）
2. 実測の数字（表形式）
3. 生成物の一覧と見方（下書き・確認リスト・承認記録）
4. 安全設計の説明（自動送信はしていない。すべて人間が確認してから送る流れ）
5. 「継続・修正・中止」の判断をお願いする一文
6. 継続する場合の次の提案（本格導入・月次サポート）の予告

条件: ですます調、A4用紙1〜2枚。効果は「見込み」「実測値」と書き分け、将来の保証はしない。

----- ここから貼り付け -----
[client_report.md の中身をここに貼る]
```

## 6. お断りメールへの返信

提案を断られたときの返信は、次の機会につなげる大事な一通です。

```text
あなたはビジネスメールの専門家です。
業務自動化の提案を断られたお客様への返信メールを作ってください。

前提:
- 宛先: [会社名] [担当者名]様
- 断られた提案: [内容。例: 請求書催促業務の自動化のお試し導入]
- 断りの理由: [先方が言った理由。例: 今は予算がない / 社内で検討したい / 必要性を感じない]
- 自分: [名前]（副業で業務自動化の支援をしている）

条件:
- まず検討へのお礼。食い下がらない、値引きもしない。
- 「状況が変わったときに思い出してもらう」ことだけをゴールにする。
- 無料で役に立つ置き土産を1つ添える（例: ヒアリングで整理した業務の課題メモはそのまま差し上げます、など）。
- 3〜6か月後に近況伺いをしてよいか、さりげなく打診する。
- ですます調で8行以内。恨みがましさゼロで。
```

## 7. トラブル時のお詫びと対応

納品物の不備や誤解が起きたときは、速さと誠実さがすべてです。

```text
あなたは顧客対応の専門家です。
業務自動化のお試し導入で問題が起きたお客様への、お詫びと対応方針のメールを作ってください。

前提:
- 宛先: [会社名] [担当者名]様
- 起きたこと: [事実。例: 納品した下書きの宛名に誤りがあった / 報告書の数字に集計ミスがあった / 期日に間に合わなかった]
- 現在の状況: [例: 外部への影響なし。下書きは送信前に止まっており、お客様から送られたものはない]
- 対応方針: [例: 本日中に修正版を納品する / 原因と再発防止をまとめて報告する]

条件:
- 構成: 事実の認め方 → お詫び → 影響範囲（外部に出ていないなら明確にそう書く）→ 対応と期日 → 再発防止 → 締め。
- 言い訳をしない。ただし「送信前に人間が確認する設計だったため外部への影響はない」など、事実としての安全設計は正確に伝える。
- 過剰な自責や無償対応の安売りはしない。約束するのは「いつまでに何をするか」だけ。
- ですます調で12行以内。
```

## 8. 継続契約の提案文

お試し導入（PoC）の報告会のあと、月次運用サポートや本格導入につなげる提案文です。

```text
あなたは継続契約の提案が得意なコンサルタントです。
業務自動化のお試し導入（PoC）が終わったお客様に、継続の提案メールを作ってください。

前提:
- 宛先: [会社名] [承認者名]様
- PoCの実測結果: 月[件数]件の[業務内容]で、1件あたり[分数]分 → 確認のみ約[分数]分に短縮の見込み
- 提案1（月次運用サポート）: 月額[金額]万円（税別）。毎月の実行と確認、下書きの調整、月次レポート、改善提案
- 提案2（本格導入）: [金額]万円（税別）。実データとの接続設計、承認ルールの整備、切り戻し手順の文書化
- 2つは別々の見積もりで、月次サポートだけでも申し込めることを明記

条件:
- 冒頭でPoCの実測数字を1行で振り返る（成果を盛らない。実測値のみ）。
- 月次サポートの「毎月やること」を箇条書きで具体的に列挙し、金額の根拠を示す。
- いつでも解約できる条件を明記する（縛りの長い契約にしない）。
- 返事の期日は切らず、「ご検討ください」で締める。
- ですます調でA4半分程度。
```

## うまく仕上がらないときのコツ

- **一度に全部直させない**: まず全体を作らせて、「2の段落だけ、もっと短く」のように部分ごとに直すと速いです。
- **実際の言葉を追加する**: ヒアリングでお客様が使った言葉（例: 「請求書の回収が遅くて困る」）をプロンプトに足すと、ぐっと刺さる文章になります。
- **最後は必ず自分の目で確認する**: 社名・金額・日付・「保証」と読める表現の4点は、送る前に必ず自分で確認してください。AIの出力をそのまま送ってはいけません。
