# Selling Business Automation Guide

This guide explains how to use AI Automation Starter Kit to propose a small business automation PoC to a company.

## Core Principle

Do not start by selling a large production system. Start with one workflow, one input, one output, one human approval point, and one success metric.

The first offer should be a dry-run PoC: the client can see the queue, drafts, approval list, report, and before/after story before any production action is enabled.

## Sales Flow

1. Choose one industry.
2. Run `flow-guide` or `opportunity-catalog`.
3. Run `business-launch` if you need a proposal system.
4. Run `beginner-sales` if you need visual demo materials.
5. Ask discovery questions.
6. Estimate conservative value.
7. Propose a small Paid PoC.
8. Run the local dry-run.
9. Generate a client report.
10. Only discuss production connectors after the client sees value and approves risk boundaries.

## Discovery Questions

- What repeated task happens every week?
- Where does the task start?
- Is the input a spreadsheet, email, form, CRM export, or chat message?
- Who reviews the final output?
- How many items happen each month?
- How many minutes does one item take?
- Where do mistakes or delays happen?
- What would make a 3-day dry-run PoC worth continuing?

## Better Sales Language

Avoid saying:

> We can automate everything with AI.

Say:

> We can start with one repeated workflow, use sample data, keep production actions disabled, and show the queue, drafts, human approval point, and report. After that, you can decide whether to continue, revise, or stop.

## Do Not Promise Income

Do not promise income, savings, or fully autonomous results. The kit helps package a professional workflow: discovery, proposal, dry-run, measurement, approval, and handoff.

## Responsible Boundaries

- Do not use client data without approval.
- Do not enable production sends without approval.
- Do not enable production writes without approval.
- Keep legal, medical, HR, finance, access, and payment decisions under client responsibility and human review.
- Split maintenance into concrete work: monitoring, fixes, monthly review, connector checks, and reporting.
