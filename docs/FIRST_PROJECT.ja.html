AI初心者の副業を、動く見本から

ダブルクリックする。まずは、それだけです。

見本データで動く業務フロー、確認画面、顧客への質問、提案資料、AIへの次の依頼文をまとめて作ります。ターミナルを覚えるところから始める必要はありません。

1. おすすめ: ファイルをダブルクリック

  1. GitHubの「Code」から「Download ZIP」を選びます。
  2. ZIPを開き、展開したフォルダの START_HERE.command をダブルクリックします。
  3. Macが確認を出した場合は、ファイルを右クリックして「開く」を選びます。
  4. ブラウザの「次はこれだけ」から進みます。

Windowsでは START_HERE_WINDOWS.bat をダブルクリックします。APIキーやクラウド契約は不要です。

2. Codexへ準備を任せる

START_WITH_AI.ja.txtを開き、全文をCodexへ渡してください。Codexが見本を準備し、完了後は一度に一問だけ質問します。顧客データやAPIキーは求めません。

3. ターミナルに慣れている場合

一行を貼り付ける方法も残しています。

git clone https://github.com/goonobu-dot/ai-automation-starter-kit.git && cd ai-automation-starter-kit && python3 scripts/first_start.py --open

4. 画面に出る4つの順番

見本を見る入力が下書きと確認記録になる流れを見ます。
一問ずつ聞くAIが顧客の困りごと、例外、承認者を確認します。
顧客向けに変える最初は見本データで変更とテストを進めます。
小さく提案する効果と停止条件を決めた有料PoCとして提案します。

5. 作られるもの

  • START_HERE.html: 人が迷わない最初の画面
  • AI_NEXT_STEP.md: CodexなどのAIエージェントに渡す次の依頼
  • 01_CLIENT_INPUT: 顧客資料を入れる前に安全条件を確認する場所
  • client_command_center.html: デモ、提案、PoC、導入判断の案内
  • 見本実行の下書き、確認記録、顧客レポート、提案書、料金検討表

6. 正直な境界

この一行だけで、顧客企業の本番システムが完成するわけではありません。
見本と副業準備までは自動で作ります。実案件では、顧客の業務、データ、例外、責任者をAIが一度に一問ずつ確認します。外部送信しません。本番更新は人が承認するまで止めます。収益を保証するものでもありません。
