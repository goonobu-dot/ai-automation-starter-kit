Manual Studio 第1段階

録画から、画像付きの業務マニュアルを作る

普段の仕事を説明しながら録画し、動画をフォルダーへ入れます。システムが音声を文字にし、画面変化と動画全体から画像候補を作ります。Codexが声と画像を合わせ、文章、注意点、例外、フロー図へ整理します。

最初にすること

顧客情報やパスワードが映っていない短い見本動画を1本用意してください。

最初は3分から10分程度の業務がおすすめです。例えば「請求書の下書きを保存する」「顧客情報を登録する」など、開始と終了が分かる仕事を選びます。

作成の流れ

1録画する
作業しながら、何を確認しているか声でも説明します。
2動画を入れる
01_RECORDINGSへ動画を入れます。
3声と画面を読む
説明を文字にし、変化した場面を動画の最後まで拾います。
4AIが整理する
画像と補足文から手順書の下書きを作ります。
5AIが質問する
不足している温度、時間、数量、判断基準などを一問ずつ聞きます。
6回答と根拠を保存
回答、資料名、回答者、日時を記録します。
7前後を比較する
AIが回答を反映し、最初の版と完成候補を並べます。
8完成版を保存
内容が正しいことを確認し、完成版フォルダーへ保存します。

実際の操作

最初の1回だけ: Codexへログインし、FFmpegとwhisper.cppを使える状態にします。MacではCodexへ「FFmpegとwhisper.cppをインストールし、このプロジェクトのManual Studioを使えるようにして」と依頼できます。APIキーは不要です。顧客情報やパスワードを含む動画は使いません。

1. 作業場所を作る

ai-automation-kit manual-studio create --output "my-manual" --name "請求入力" --language ja

2. 動画を入れる

my-manual/01_RECORDINGSを開き、音声付きの画面録画を入れます。「このボタンを押します」「ここで金額を確認します」のように、判断の理由も声で説明すると詳しい手順になります。

3. 音声を文字にして、重要画面の候補を作る

--transcribeを付けると、動画の声をパソコン内で文字にして03_TRANSCRIPTSへ保存します。初回だけ多言語モデル（small、約466MB）を取得し、照合後は同じモデルを再利用します。すでに同名の文字起こしがある場合は上書きしません。

ai-automation-kit manual-studio prepare --workspace "my-manual" --transcribe

音声がない動画では--transcribeを外してください。自分で直した文字起こしを使う場合は、動画と同じ名前の.txtを03_TRANSCRIPTSへ入れてから通常の準備を実行します。

4. Codexで画像付きマニュアルを作る

ai-automation-kit manual-studio build --workspace "my-manual" --title "請求入力" --open

5. AIに不足情報を一つずつ聞いてもらう

Codexへ次のように依頼できます。

このManual Studioの不足質問を一つずつ聞いてください。
私の回答と根拠を保存し、完成前プレビューまで進めてください。

質問だけを確認する場合は、次の処理を実行します。

ai-automation-kit manual-studio questions --workspace "my-manual"

6. 回答と根拠を保存する

温度、時間、数量、機械設定、安全基準はAIに推測させません。正式な資料名、または業務責任者が確認したことを根拠として保存します。

ai-automation-kit manual-studio answer --workspace "my-manual" \
  --answer "220度で18分" \
  --source-kind document \
  --source "工程標準書B-12 2026年4月版" \
  --answered-by "製造責任者"

今は分からない場合は--answerの代わりに--deferを使い、--sourceへ保留理由、--answered-byへ確認した人を入れます。保留が残っている間は完成処理が止まります。

7. 回答を反映して完成候補を作る

ai-automation-kit manual-studio complete --workspace "my-manual" --title "請求入力" --open

AIが新しい不足を見つけた場合は、無理に完成扱いにせず、質問画面へ戻ります。不足がなければ、回答前と回答後を並べた比較画面が開きます。

8. 確認した内容だけを完成版として保存する

ai-automation-kit manual-studio approve --workspace "my-manual" \
  --approved-by "業務責任者" --open

完成版は06_APPROVED_MANUALへ保存され、承認者、日時、ファイル照合値も記録されます。

自動で作られるもの

画像付きHTML

各手順にスクリーンショット、説明、動画名、時刻、注意点を表示します。

ローカル文字起こし

動画の説明音声をパソコン内でテキスト化し、元動画と使用モデルの記録も残します。

フロー図

作業の順番をMermaid形式で保存し、後から編集できます。

確認チェックリスト

画像の一致、秘密情報、例外、別の人による再現を確認します。

自動化候補

録画から確認できた作業と、不足している質問を分けて保存します。

回答ガイド

AIが次に聞く質問、回答数、保留数をHTMLで表示します。

回答台帳

回答、根拠、回答者、日時をJSONへ保存し、下書きが変わった場合は混同しないよう検知します。

完成前の比較

最初のマニュアルと回答反映後を左右に並べ、変更内容を確認できます。

完成版

確認したHTML、手順データ、承認記録を06_APPROVED_MANUALへ保存します。

今回まだ自動化しない部分

  • 雑音、固有名詞、早口では誤認識が起こります。完成前に文字と画像が合っているか確認します。
  • AIが選んだ画像と、回答が反映された文章は完成版へ保存する前に確認します。
  • 根拠がない温度、時間、数量、設定値はAIが推測せず、質問を未解決として残します。
  • 回答資料だけで追加された工程には、無関係な動画画像を付けません。「回答根拠」と質問番号を表示します。
  • 作成した手順書から、実際の顧客システムを自動操作することはまだ行いません。
  • 外部送信、公開、支払い、承認は実行しません。
まずは短い見本動画で、画像と文章が正しく対応するところまで確認してください。
