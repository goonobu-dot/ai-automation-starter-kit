# Grill Me プロンプト集

このページは、AI初心者がそのままコピペできる相談文集です。ポイントは、AIに一気に答えを出させるのではなく、1問ずつ確認させることです。

## 案件選定

```text
私はAI初心者です。
企業向けの小さな業務自動化PoCを作りたいです。
どの業務から始めるべきか、1問ずつ質問してください。

判断基準は、説明しやすいこと、サンプルデータで試せること、人間の承認を残せること、危険な自動実行を避けられることです。
```

## 顧客ヒアリング

```text
これから顧客に業務自動化のヒアリングをします。
顧客ヒアリングで聞くべき質問を、1問ずつ出してください。

最初に聞くのは、現在困っている作業、入力元、出力先、確認者、失敗時の影響です。
顧客が曖昧に答えた場合は、具体例を求める追加質問を作ってください。
```

## フロー選定レビュー

```text
このフローが顧客に合っているか確認してください。
いきなり結論を出さず、1問ずつ確認してください。

確認したいこと:
- 業務の痛み
- サンプルデータ
- 出力先
- 人間の承認
- 成功指標
- 自動化してはいけないこと
```

## クラウド準備 / cloud-plan 確認

```text
cloud-plan の出力を読んでください。
私はクラウド初心者です。

次に確認すべきことを1問ずつ質問してください。
課金、secret、IAM、domain、webhook、scheduler、queue、log、rollback、人間の承認を分けて確認してください。
本物のAPIキーやsecretはチャットに貼らせないでください。
```

## API / secret / webhook 確認

```text
API、secret、webhook の設定で止まっています。
本物の値は貼りません。

どの情報なら安全に共有できるかを教えてから、1問ずつ確認してください。
secret名、エラー文、設定画面の項目名、ログの有無を分けて確認してください。
```

## 提案レビュー

```text
この自動化提案を顧客に出す前に、厳しくレビューしてください。

1問ずつ確認してください。
完全自動化、売上保証、無人運用、危険な本番更新を約束していないか確認してください。
PoCの範囲、費用、期間、成果物、承認者、停止条件が明確か確認してください。
```

## 危険な自動化を止める

```text
この自動化案に危険がないか確認してください。

医療、法律、金融、個人情報、本番送信、契約、支払い、値引き、削除、解約、顧客への約束に関わる部分があれば止めてください。
安全なdry-runまたは承認キューに変える方法を1問ずつ確認してください。
```
