Manual Studio, first stage

Turn a recording into an illustrated work manual

Explain the work while recording it, then add the video to a folder. The system transcribes speech locally and captures screenshots across the full recording. Codex combines the voice and images into instructions, warnings, exceptions, and a workflow diagram.

Start here

Add a recording that contains no client information or passwords.

Begin with a three-to-ten-minute task with a clear start and finish, such as saving an invoice draft or registering a sample customer.

Work through it with Codex

You do not need to type terminal commands. Tell Codex what you want in ordinary language. Codex can check the required tools, create the workspace, and open each screen. No API key is required. The first speech-recognition setup may take a little longer.

1. Ask Codex to prepare the workspace

Open this project with Codex and say:

I want to use Manual Studio in this project to create an illustrated work manual. Check what is needed, create the workspace, open the place where I should add the recording, and guide me one step at a time.

2. Add the recording

Add a recording with speech to the location Codex opens. Explain both the action and the reason, such as “Press this button” and “Check the amount here.”

3. Ask Codex to read the voice and screens

I added the recording. Transcribe the speech and create candidate images for the instructions.

The first run may take longer while local speech recognition is prepared. Codex handles the setup and storage.

4. Open the illustrated draft

Review the recording and transcript, create the illustrated draft manual, and open it in my browser.

5. Choose the clearest image for each step

Open the screen for choosing better step images.

Candidate frames appear in recording order. Choose a sharp image where hands do not hide the important area and the before-or-after state is easy to understand, then select “Save selected images.”

6. Confirm missing details one question at a time

Ask the Manual Studio questions one at a time. Save my answers and sources, then continue to the final review screen.

7. Give each answer and its source

Include the answer, the document or person that supports it, and the responder. For example: “220 C for 18 minutes. The source is Process Standard B-12, confirmed by the production lead.” If the answer is unknown, ask Codex to defer it and record who should confirm it. The workflow will not treat an unresolved value as completed.

8. Compare the before-and-after versions

Apply the saved answers and show the original draft beside the completed candidate.

If Codex finds another gap, it returns to the questions. Otherwise, the comparison screen opens.

9. Save the reviewed manual

I reviewed the content. Save the completed manual with “Process owner” as the reviewer, then open it.

The completed manual also records the reviewer, save time, and integrity information.

Operators who need internal commands or troubleshooting details can use the full operating manual. Beginners do not need to read it first.

Generated artifacts

Illustrated HTML

Each step shows its screenshot, instruction, source video, timestamp, and warning.

Image picker

Compare candidate frames for each step in recording order and save the clearest one into the manual.

Local transcript

The spoken explanation is converted to text on the computer, with provenance for the source video and model.

Workflow diagram

The ordered steps are saved as an editable Mermaid diagram.

Review checklist

Reviewers check image accuracy, secrets, exceptions, and whether another person can follow the manual.

Automation candidate

Confirmed work and unresolved questions are kept separate before implementation.

Answer guide

An HTML page shows the next AI question and the answered, pending, and deferred counts.

Answer ledger

Answers, sources, responders, and timestamps are stored in JSON and bound to the current draft.

Before-and-after review

The original manual and answer-completed candidate are displayed side by side.

Completed manual

The reviewed HTML, structured plan, and review record are collected in the completed-manual location.

Not automated in this first stage

  • Noise, names, and fast speech can be misrecognized. Verify that the final words and images agree.
  • AI-selected screenshots are candidates. Use the image picker to verify that hands, work items, and results are clear, then replace weak frames.
  • AI leaves unsupported temperatures, durations, quantities, and settings unresolved instead of guessing.
  • A step added only from recorded answers remains image-free and cites its answer question IDs instead of displaying an unrelated screenshot.
  • The draft does not operate a real client system.
  • No external send, publishing, payment, or approval action is performed.
Start with a short sample recording and verify that every screenshot matches its instruction.
