Manual Studio, first stage

Turn a recording into an illustrated work manual

Explain the work while recording it, then add the video to a folder. The system transcribes speech locally and captures screenshots across the full recording. Codex combines the voice and images into instructions, warnings, exceptions, and a workflow diagram.

Start here

Add a recording that contains no client information or passwords.

Begin with a three-to-ten-minute task with a clear start and finish, such as saving an invoice draft or registering a sample customer.

Commands

One-time setup: Sign in to Codex and make FFmpeg and whisper.cpp available. On a Mac, ask Codex: “Install FFmpeg and whisper.cpp, then prepare this project for Manual Studio.” No API key is required. Do not use a recording that contains client data or passwords.

1. Create the workspace

ai-automation-kit manual-studio create --output "my-manual" --name "Invoice entry" --language en

2. Add the recording

Open my-manual/01_RECORDINGS and add a recording with speech. Explain both the action and the reason, such as “Press this button” and “Check the amount here.”

3. Transcribe speech and extract important screens

--transcribe converts speech to text locally and saves it in 03_TRANSCRIPTS. The first run downloads and verifies the multilingual small model (about 466 MB); later runs reuse it. An existing matching transcript is never replaced.

ai-automation-kit manual-studio prepare --workspace "my-manual" --transcribe

Remove --transcribe for a silent recording. To use a corrected transcript, place a matching .txt file in 03_TRANSCRIPTS before running the normal prepare command.

4. Build and open the illustrated manual

ai-automation-kit manual-studio build --workspace "my-manual" --title "Invoice entry" --open

5. Ask AI to collect missing details one question at a time

You can ask Codex:

Ask the Manual Studio questions one at a time.
Save my answers and sources, then continue to the final preview.

To inspect the next question directly:

ai-automation-kit manual-studio questions --workspace "my-manual"

6. Save an answer and its source

Do not let AI guess temperatures, durations, quantities, machine settings, or safety rules. Save an official document or a process-owner confirmation as the source.

ai-automation-kit manual-studio answer --workspace "my-manual" \
  --answer "220 C for 18 minutes" \
  --source-kind document \
  --source "Process standard B-12, April 2026" \
  --answered-by "Production lead"

Use --defer instead of --answer when the fact is unknown, put the reason in --source, and name the person who checked in --answered-by. Completion remains blocked while a deferred question exists.

7. Apply the answers and create the final candidate

ai-automation-kit manual-studio complete --workspace "my-manual" --title "Invoice entry" --open

If Codex finds another gap, the workflow returns to questions instead of pretending the manual is complete. Otherwise, a side-by-side before-and-after review opens.

8. Save only the reviewed candidate as completed

ai-automation-kit manual-studio approve --workspace "my-manual" \
  --approved-by "Process owner" --open

The completed HTML, structured plan, approver, time, and integrity hashes are saved in 06_APPROVED_MANUAL.

Generated artifacts

Illustrated HTML

Each step shows its screenshot, instruction, source video, timestamp, and warning.

Local transcript

The spoken explanation is converted to text on the computer, with provenance for the source video and model.

Workflow diagram

The ordered steps are saved as an editable Mermaid diagram.

Review checklist

Reviewers check image accuracy, secrets, exceptions, and whether another person can follow the manual.

Automation candidate

Confirmed work and unresolved questions are kept separate before implementation.

Answer guide

An HTML page shows the next AI question and the answered, pending, and deferred counts.

Answer ledger

Answers, sources, responders, and timestamps are stored in JSON and bound to the current draft.

Before-and-after review

The original manual and answer-completed candidate are displayed side by side.

Completed manual

The reviewed HTML, structured plan, and approval record are saved in 06_APPROVED_MANUAL.

Not automated in this first stage

  • Noise, names, and fast speech can be misrecognized. Verify that the final words and images agree.
  • Review every AI-selected screenshot and every answer-completed instruction before saving the completed manual.
  • AI leaves unsupported temperatures, durations, quantities, and settings unresolved instead of guessing.
  • A step added only from recorded answers remains image-free and cites its answer question IDs instead of displaying an unrelated screenshot.
  • The draft does not operate a real client system.
  • No external send, publishing, payment, or approval action is performed.
Start with a short sample recording and verify that every screenshot matches its instruction.
