# Grill Me チェックリスト

このチェックリストは、AIに初心者を1問ずつ確認させるための確認項目です。自分で全部判断できなくても、AIにこの項目を使って質問してもらえば前に進めます。

## 業務適合

- 業務の痛みは明確か。
- 入力元は明確か。
- 出力先は明確か。
- サンプルデータで試せるか。
- 成功指標を数字で言えるか。
- 人間の承認を残せるか。
- 自動化しない範囲が明確か。

## 顧客ヒアリング

- 誰が現在作業しているか。
- どれくらい時間がかかっているか。
- どのツールを使っているか。
- どのデータをAIが読んでよいか。
- どの出力をAIが作ってよいか。
- 誰が承認するか。
- 失敗した時に誰が止めるか。

## ローカル dry-run

- 本番送信しない設定か。
- サンプルデータだけを使っているか。
- work queue が生成されるか。
- draft output が生成されるか。
- approval queue が残るか。
- 顧客に説明できる before/after があるか。

## クラウド準備

- cloud にする理由があるか。
- workload は決まっているか。
- provider は決まっているか。
- billing owner はいるか。
- secret の保存場所は決まっているか。
- logs を見る人は決まっているか。
- rollback owner はいるか。
- 本番 traffic を有効にする承認者はいるか。

## 人間の承認

- 顧客への送信前に承認があるか。
- CRMやspreadsheet更新前に承認があるか。
- webhook有効化前に承認があるか。
- scheduler有効化前に承認があるか。
- queue consumer有効化前に承認があるか。
- domainやpublic URL公開前に承認があるか。

## 停止条件

- secret が足りない。
- logs が見えない。
- 顧客が本番送信を急がせる。
- 成功指標が決まっていない。
- 顧客がAPI権限を理解していない。
- rollback owner がいない。
- 個人情報や高リスク判断が混ざっている。

この停止条件に当てはまる場合は、先に進まず dry-run または再設計に戻します。
