A working example for an AI automation side business

Double-click. Start there.

Create a workflow that runs on sample data, a review screen, client questions, proposal assets, and the next AI-agent prompt. You do not need to learn terminal commands first.

1. Recommended: double-click a file

  1. Choose “Code” and “Download ZIP” on GitHub.
  2. Open the ZIP and double-click START_HERE.command on macOS.
  3. On Windows, double-click START_HERE_WINDOWS.bat.
  4. Continue from “Do only this next” in the browser.

No API key or cloud account is required. On macOS, if the first launch is blocked, right-click the file and choose Open.

2. Let Codex prepare it

Give START_WITH_AI.txt to Codex. It will prepare the example and then ask you one question at a time without requesting client data or API keys.

3. Terminal fallback: Paste one line

git clone https://github.com/goonobu-dot/ai-automation-starter-kit.git && cd ai-automation-starter-kit && python3 scripts/first_start.py --open --language en

5. What it creates

  • START_HERE.html: the human-friendly first screen
  • AI_NEXT_STEP.md: the next request for Codex or another AI coding agent
  • 01_CLIENT_INPUT: a boundary check before client files are added
  • client_command_center.html: demo, proposal, pilot, and go-live navigation
  • Sample drafts, approval evidence, a client report, proposal assets, and pricing worksheets

6. Honest limits

This one line does not create a finished production system for a client.
It creates a working example and side-business preparation. For a real client, AI asks one question at a time about the workflow, data, exceptions, and owner. It does not send anything externally, and production changes remain blocked until a human approves them. It does not guarantee income.
