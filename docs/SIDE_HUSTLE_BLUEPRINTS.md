# Side Hustle Blueprints Guide

This guide explains the `side-hustle-blueprints` command.

Use it when you want to choose a sellable small business automation offer before building a custom project.

## What It Creates

```bash
ai-automation-kit side-hustle-blueprints --industry local-business --operator-level beginner --output .tmp/side-hustle-blueprints
```

Important files:

- `START_HERE_SIDE_HUSTLE_BLUEPRINTS.md`: the first file to read
- `side_hustle_blueprints.md`: catalog of sellable automation offers
- `side_hustle_blueprints.html`: browser-friendly catalog
- `side_hustle_blueprints.csv`: spreadsheet-friendly catalog
- `first_client_picker.md`: scoring sheet for choosing the easiest first client
- `client_intake_questions.md`: questions to ask before proposing automation
- `implementation_paths.md`: local dry-run, spreadsheet-first, workflow platform, AI agent, and cloud paths
- `offer_scripts.md`: simple outreach and discovery scripts
- `risk_boundaries.md`: what must stay behind human approval
- `ai_agent_handoff.md`: copy-ready instruction for Codex, Claude Code, Cursor, ChatGPT, Claude, or another AI agent

## Why This Matters

Many beginners start with "I can automate anything." That is too broad to sell safely.

This command turns broad automation ideas into concrete offers such as:

- website plus inquiry operations
- AI reception and first reply
- invoice and document follow-up
- internal FAQ assistant
- sales research brief
- review response desk
- appointment reminder workflow
- lead routing and follow-up
- report-to-dashboard pack
- document intake and filing desk
- inventory and reorder alerts
- client onboarding portal lite

Each offer includes buyer type, pain, required inputs, deliverables, tools, price range, human approval boundary, and a first pitch.

## Beginner Rule

Do not sell "fully automatic business replacement."

Sell a bounded proof of value:

1. collect approved sample data
2. map the current workflow
3. generate a small dry-run or spreadsheet-first workflow
4. review outputs with a human
5. measure whether it saves time or reduces missed work
6. only then discuss production automation

## Using It With An AI Agent

Open `ai_agent_handoff.md` and give it to your AI agent. Ask the agent to interview you one question at a time.

Good prompt:

> Read this GitHub project and the generated `side-hustle-blueprints` folder. Help me choose one beginner-friendly automation offer. Ask one question at a time. Do not ask for secrets or private customer data.

## Public Source Inspiration

The blueprints are inspired by public workflow automation, AI app builder, agent, browser automation, and local dry-run patterns. Public projects can teach patterns and integrations, but they are not permission to copy brands, private data, or proprietary workflows.
