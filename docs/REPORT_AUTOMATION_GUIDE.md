# Report Automation Guide

This guide explains how to sell and deliver folder-based daily, weekly, or monthly report drafting for small businesses.

The goal is not fully autonomous submission.

The goal is safer draft automation:

```text
Store past completed reports
Store current materials
Let AI infer structure and style from past outputs
Let AI draft from current evidence
Ask one GrillMe-style question at a time when facts are unclear
Keep final submission behind human approval
```

## Command

```bash
ai-automation-kit report-automation \
  --report-type monthly \
  --client-type local-business \
  --niche construction \
  --output .tmp/monthly-report-pack
```

Open these files first:

```text
.tmp/monthly-report-pack/START_HERE_REPORT_AUTOMATION.md
.tmp/monthly-report-pack/demo_report_automation.html
.tmp/monthly-report-pack/ai_agent_prompt.md
```

## Workspace

| Folder | Purpose |
|---|---|
| `01_past_outputs/` | Past completed daily, weekly, and monthly reports |
| `02_current_materials/` | Current CSVs, notes, photos, attachments, and task logs |
| `03_templates/` | Daily and monthly report templates |
| `04_ai_analysis/` | Writing style profile, missing facts, and contradictions |
| `05_grill_me_questions/` | One-question-at-a-time clarification loop |
| `06_drafts/` | AI draft reports |
| `07_approval/` | Human approval checklist |

## Why It Is Useful

Most small teams already create repeated reports, but the work is manual and easy to delay. This pack gives a beginner-friendly service offer:

- Organize report source folders.
- Use past completed reports as style evidence.
- Draft the new report locally.
- Ask questions instead of inventing missing facts.
- Let a human approve before submission.

## Good First Offers

- Construction daily report drafting
- Store daily and monthly sales report drafting
- Hotel or tourism monthly operations report drafting
- Sales weekly report drafting
- Manufacturing work-log summary
- Social media monthly performance report
- Inquiry handling monthly summary

## Sales Script

```text
We can create a local dry-run system where past completed reports and current materials are placed into folders.
AI drafts the next report from those sources.
If something is unclear, it asks one question at a time instead of inventing the answer.
The final report stays behind human approval.
```

## Do Not Promise

- Fully automatic final submission
- Guaranteed revenue or savings
- AI-only cause analysis
- Human-free approval
- Production cloud syncing before dry-run review
