# 初心者向けクラウド挑戦プレイブック

このプレイブックは、AI初心者が「これなら自分も企業向け自動化に挑戦できそう」と感じるための実践ルートです。目的は、いきなり大きなシステムを作ることではありません。最初の有料PoCとして、1つの業務、1つの入力、1つの出力、1人の承認者に絞って、小さく価値を見せることです。

## まず目指す状態

最初のゴールは、次の状態です。

- 企業の困りごとを1つに絞る。
- その業務に合うフローを1つ選ぶ。
- ローカル dry-run で見本を作る。
- 顧客に before/after を説明できる。
- クラウド化に必要な情報を一覧化できる。
- AIができることと、人間が承認することを分けられる。
- 小さな有料PoCとして提案できる。

## 用意するもの

最低限、以下があれば始められます。

- パソコン。
- GitHub からこのプロジェクトを取得できる環境。
- Python を実行できる環境。
- 自動化したい業務の説明。
- サンプルデータ。CSV、メール例、問い合わせ例、スプレッドシート例など。
- 結果を誰が確認するか。
- 本番送信してよい条件。
- 顧客が使っているツール。Gmail、Google Sheets、Slack、LINE、CRM、共有フォルダなど。

本物の APIキー、secret、顧客の機密情報は、最初からチャットに入れないでください。

## AIができること

- 業務フローを候補から選ぶ。
- `complete-workspace` で説明用の見本を作る。
- `guided-setup` で必要な質問を整理する。
- `guided-review` で不足を洗い出す。
- `cloud-plan` でクラウド導入資料を作る。
- 顧客説明文、提案文、作業範囲、費用の考え方を作る。
- 失敗時の rollback 手順を作る。

## 人間が承認すること

- 顧客のデータを使ってよいか。
- どのクラウドアカウントを使うか。
- 誰が課金を管理するか。
- どのAPI権限を許可するか。
- 本番送信や本番webhookを有効にしてよいか。
- 失敗時に誰が止めるか。

## 最初の有料PoCの作り方

提案の形はシンプルで十分です。

1. 現在の業務を聞く。
2. その業務に合うフローを1つ選ぶ。
3. サンプルデータだけで dry-run を作る。
4. 生成された work queue、draft output、approval queue を見せる。
5. 手作業のどこが減るかを説明する。
6. 3日から2週間程度の小さなPoCとして提案する。
7. 本番化は別判断にする。

最初から「完全自動化」「24時間無人運用」「売上保証」を言わないでください。初心者が信頼を取るには、できる範囲を正確に切ることが重要です。

## 顧客に見せる言い方

「この仕組みは、いきなり全部を自動化するものではありません。まずはサンプルデータで業務の流れを再現し、AIが下書きや作業キューを作り、人間が確認する形にします。価値が見えたら、クラウドで動かすために必要なAPI、secret、費用、権限、停止手順を整理します。」

この説明なら、AI初心者でも安全に提案できます。

## 最初に選びやすい業務

- 請求書や書類のフォロー。
- 問い合わせ返信の下書き。
- 予約リマインド。
- 週次レポート作成。
- 顧客リストやCRMの整理。
- 在庫アラート。
- 面談日程調整。
- 社内FAQの振り分け。

これらは、入力と出力が比較的分かりやすく、人間の承認を残しやすい業務です。

## 最初に避ける業務

- 医療、法律、金融判断など高リスクな判断。
- 自動で契約、支払い、値引き、解約を実行する業務。
- 大量の個人情報を扱う業務。
- 複数部署をまたぐ大規模導入。
- 失敗時の停止方法が決まっていない業務。

## 1週間の進め方

Day 1:
業務を1つ選び、サンプルデータを集める。

Day 2:
`complete-workspace` を実行し、ローカルの見本を作る。

Day 3:
dry-run の出力を確認し、顧客に見せる画面や資料を整える。

Day 4:
顧客に before/after を説明し、足りない情報を聞く。

Day 5:
`guided-setup` と `guided-review` で不足を整理する。

Day 6:
`cloud-plan` でクラウド導入資料を作る。

Day 7:
有料PoCとして、範囲、費用、期間、承認者、停止条件を提案する。

## 成功の考え方

初心者が最初に勝つポイントは、難しいクラウド設定を全部覚えることではありません。顧客に対して、何を用意すれば自動化に近づくのか、どこは人間が確認するのか、どこから費用が発生するのかを分かりやすく説明することです。

このプロジェクトは、その説明と準備をAIに手伝わせるためのものです。
