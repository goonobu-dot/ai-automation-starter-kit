AI AUTOMATION STARTER KIT — OPERATING MANUAL / v0.1.0

操作マニュアル中小企業への業務自動化提案を、コマンドで支度する

このページは、ブラウザで読める操作マニュアルです。AIエージェント（指示を受けて作業を進めてくれるAIのこと）を触り始めたばかりの方が、副業の最初の1案件を安全に完了するために必要なコマンドだけをまとめています。出力例はすべて実際に動かして得たものです。

生成する人間が確認・承認人間が送る

このキットは外部に何も送信しません。すべての出力はドライラン（本番実行しないお試し実行）の「下書き」です。収益を保証するものではなく、本ページの金額はすべて相場の目安です。

第 1 節副業の5段階ロードマップ

最も簡単な入口: ZIPを展開し、Macは START_HERE.command、Windowsは START_HERE_WINDOWS.bat をダブルクリックしてください。Codexへ任せる場合は START_WITH_AI.ja.txt を渡します。詳しくはダブルクリック手順書を開いてください。

詳しく選んで進めたい場合の案内役は ai-automation-kit beginner です。最初の1案件までの道のりを、5つの駅に分けて案内します。いまの駅を選ぶと「やること・コマンド・開くファイル・次の一歩」が表示されます。迷子になったら、いつでもこのコマンドに戻ってください。

  1. 1 環境準備 キットを入れて、正しく動く状態にする pip install -e .doctor はじめかた を読みながら約5分
  2. 2 最初のデモ お客様に見せられるデモ一式を自分のPCに作る complete-workspace 生成された demo_site/index.html をブラウザで開く
  3. 3 営業準備 ヒアリングシート・提案書・価格メニューを用意する beginner-salesoffer-pack チュートリアル 第1〜5章がこの駅です
  4. 4 最初の案件実行 ドライラン → 目視確認 → 承認、まで動かす flows installflows runflows approve 送信は必ず人間の手で（第4節）
  5. 5 納品と請求 報告書と納品ZIPを渡し、請求までつなげる client-reportpackage-client-demoshare-check 2件目はこの路線を駅3から繰り返します

この路線図は、副業で最初の1案件を完了するまでの5段階のロードマップです。beginner コマンドの5段階と同じ並びなので、いまの駅の番号を --step に入れれば次の一歩が分かります。1案件を終えたら、駅3（営業準備）に戻って2件目を繰り返します。

第 2 節インストールと診断

インストール

git clone https://github.com/goonobu-dot/ai-automation-starter-kit.git
cd ai-automation-starter-kit
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip setuptools
python3 -m pip install -e .
ai-automation-kit --version
ai-automation-kit 0.1.0

venv はこのプロジェクト専用の仮想環境（他の環境を汚さない作業部屋）です。Windows (PowerShell) では4行目を .venv\Scripts\Activate.ps1 に読み替えてください。

doctor — 環境診断

Pythonのバージョン・書き込み権限・パッケージ導入状態などを診断し、NG項目には日本語の対処法を出します。エラーで困ったら、まずこれです。

ai-automation-kit doctor --output .tmp/doctor
status=warning
report=.tmp/doctor/doctor_report.md

ready＝問題なし、warning＝動くが注意あり（GITHUB_TOKEN未設定など。無視して構いません）、blocked＝レポートの対処法に従って直してから先へ。

complete-workspace — デモ一式のワンコマンド生成

デモサイト・提案資料・報告書・納品チェックリストまで、商談に必要な一式を1コマンドで生成します。

completion_checklist=.tmp/complete-accounting/completion_checklist.md
client_demo_package=.tmp/complete-accounting/client_demo_package/client_demo_package.zip
status=ready_to_share

demo_site/index.htmlお客様に見せるデモ画面
FINAL_DELIVERY_GUIDE.md生成物をどの順で使うかの1枚ガイド

第 4 節flows — 案件実行

72種類の業務フロー（作業手順のセット）を「一覧 → 導入 → 検証 → ドライラン → 承認」の順に操作します。

ここが安全設計の心臓部です。 run は下書きと承認待ちリストを作るだけ、approve は「人間が確認した」記録を残すだけです。実際の送信は、承認後の下書きを人間が本物のメールソフト等にコピーして行います。自動送信の機能は存在しません（仕様です）。

AIキットが自動でやること（下書き作りまで）

フローを選ぶflows list / show
→
導入flows install
→
ドライラン実行flows run
→
下書き・承認待ちリスト生成draft_outputs.md / approval_queue.csv

ここから先は、必ず人間の仕事です

人間がやること（外に出すのは必ず人）

下書きを目視確認問題があれば直して再実行
→
承認を記録flows approve
→
ローカル送信箱local_outbox/（まだ未送信）
→
人間が送信本物のメールソフト等にコピーして送る

この図は自動化の仕組みの全体像です。上の枠（AIキット）が作るのは「下書きと確認リスト」までで、外部への送信機能はそもそも存在しません。下の枠のとおり、確認・承認して外に出すのは必ず人間です。だから初心者が試しても、勝手にメールが飛ぶ事故は起きません。

ai-automation-kit flows list --industry finance   # 一覧（全部見るなら引数なし）
invoice-document-followup   finance  document  Invoice and Document Follow-up
expense-policy-check        finance  approval  Expense Policy Check
…
count=7
ai-automation-kit flows show invoice-document-followup      # 詳細と承認ポイント
ai-automation-kit flows diagram invoice-document-followup --output .tmp/diagram
flow_diagram=.tmp/diagram/flow_diagram.html   # お客様向けのフロー図解HTML
ai-automation-kit flows install invoice-document-followup --output .tmp/first-job
ai-automation-kit flows validate .tmp/first-job              # 実行前の点検
status=ready
ai-automation-kit flows run .tmp/first-job                  # ドライラン実行
automation_status=succeeded
rows_processed=1
draft_outputs=.tmp/first-job/automation_output/draft_outputs.md
approval_queue=.tmp/first-job/automation_output/approval_queue.csv
# ↑ draft_outputs.md を必ず目で読んでから ↓
ai-automation-kit flows approve .tmp/first-job --approver yamada@example.com
approval_status=approved
approved_items=2
outbox=.tmp/first-job/local_outbox/email_drafts.md

--approver は承認した人の名前かメールアドレス（誰が確認したかの記録）。案件では sample_data/input.csv をお客様のマスキング済みデータの形式に合わせてから回します。

第 5 節納品コマンド

client-report — お客様向け報告書

ai-automation-kit client-report --flow-project .tmp/first-job --output .tmp/delivery
client_report=.tmp/delivery/client_report.md
status=ready

実行結果の要約と「継続・修正・中止」の判断材料が入ります。日本語への仕上げは AI用プロンプト集 の「納品報告書の文章化」で。

package-client-demo — 納品用ZIP

ai-automation-kit package-client-demo --source .tmp/first-job --output .tmp/delivery-package
client_demo_package=.tmp/delivery-package/client_demo_package.zip
file_count=23

share-check — 共有前の安全確認

ai-automation-kit share-check --source .tmp/delivery-package --output .tmp/share-check
share_check=.tmp/share-check/share_check.md
status=ready
送る前に必ず実行。 秘密情報らしき文字列やローカルパスの混入を点検します。status=blocked のときは、直すまで絶対に共有しないでください。

beginner-sales — 初心者向け営業パック

ai-automation-kit beginner-sales --flow-id invoice-document-followup \
  --client-type local-business --niche accounting --output .tmp/beginner-sales
proposal=.tmp/beginner-sales/proposal_one_pager.md

client_delivery_checklist.md納品チェックリスト

offer-pack — 提案書・SOWの正式セット

提案書・作業範囲記述書（SOW＝やること/やらないことの合意書）・価格モデル・リスク境界を生成します。

ai-automation-kit offer-pack --business-area operations --client-type small-business \
  --source-output .tmp/complete-accounting --output .tmp/offer-pack
proposal=.tmp/offer-pack/proposal.md
statement_of_work=.tmp/offer-pack/statement_of_work.md
status=ready

business-launch — 開業スターターパック

「何を売るか」から決めたい人向け。推奨フローの選定と最初のオファー定義を生成します。

ai-automation-kit business-launch --industry finance --client-type local-business \
  --niche accounting --output .tmp/business-launch
recommended_flow=invoice-document-followup
status=ready

report-automation — 日報・月報AI下書き自動化

過去の完成物と今回資料をフォルダに入れ、AIが下書きを作り、不明点だけGrillMe方式で1問ずつ確認します。最後の提出は人間が承認します。

ai-automation-kit report-automation --report-type monthly \
  --client-type local-business --niche construction --output .tmp/monthly-report-pack
report_automation=.tmp/monthly-report-pack/START_HERE_REPORT_AUTOMATION.md
grill_me_questions=.tmp/monthly-report-pack/05_grill_me_questions/questions.md
approval_checklist=.tmp/monthly-report-pack/07_approval/approval_checklist.md
status=ready

詳しくは 日報・月報AI下書き自動化ガイド を開いてください。

レポートウィザードの再開可能な確認フローは、ai-automation-kit report-wizard serve --workspace .tmp/report-wizard-weekly --language ja --no-open で開き、レポートウィザード手順書 を見ながら進めてください。

第 7 節商談の流れ（チュートリアル要約）

営業から請求までの全工程は 中小企業への自動化提案チュートリアル に、各章「使うコマンド・実出力例・よくある失敗」つきでまとまっています。ここでは骨子だけ示します。

1営業先探し→ 2ヒアリング→ 3無料デモ→ 4提案・見積→ 5受注・契約→ 6案件実行→ 7納品→ 8請求・継続

この図は1案件の流れです。1〜5が商談の前半（営業系コマンドの出番）、6〜8が後半（flows と納品コマンドの出番）です。断られたら1に戻り、継続が決まれば6〜8を毎月繰り返します。

  1. 営業先探しと声かけまず知人・接点のある会社から。outreach_messages.md の文例をそのまま使う
  2. ヒアリングclient_questions.md を読み上げる。★印（月の件数・1件の時間・承認者・成功条件）は必ずメモ
  3. 無料デモcomplete-workspace で生成。見せるのは demo_site と before/after 比較の2〜3画面だけ
  4. 提案書と見積もりoffer-pack。価格は削減価値から逆算（お試し導入は価値の1〜2か月分が目安）
  5. 受注と契約SOWで範囲・免責・秘密情報の扱いを書面合意。実データは預からず、マスキング済みサンプルで
  6. 案件実行flows install → run → 目視確認 → approve。送信は人間の手で（第4節）
  7. 納品client-report と納品ZIP。share-check を通してから渡し、30分の報告会をセットに
  8. 請求と継続提案報告会後すぐ請求書を発行。効果が数字で見えたら、本格導入と月次サポートを別見積もりで提案

第 8 節困ったとき

症状対処
command not found仮想環境を有効化（source .venv/bin/activate）。駄目ならリポジトリ直下で pip install -e . を再実行
コマンドがエラーで止まるdoctor --output .tmp/doctor を実行し、レポートの「対処法（日本語）」に従う
flows run が失敗するflows validate で不足ファイルを確認。壊れていたら flows install からやり直す
フローIDが分からないflows list で確認（IDはハイフン区切りの英語）
共有してよいか不安share-check を実行。blocked なら共有しない
次に何をすればいいか迷子ai-automation-kit beginner に戻る
安全に関する約束。 APIキー・パスワード・お客様の実データは、チャットにも生成ファイルにも書かないでください。生成物は「下書き」です。外部への送信・本番反映は、必ず人間が確認してから人間の手で行います。
