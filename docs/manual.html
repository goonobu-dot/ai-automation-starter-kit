AI AUTOMATION STARTER KIT — OPERATING MANUAL / v0.1.0

Operating Manual

A browser-friendly manual for the core commands. This is the concise English edition; the Japanese edition (manual.ja.html) is the full version, and the Markdown manuals (English / Japanese) contain every detail. All output examples were captured from real runs.

The kit draftsA human reviews & approvesA human sends

Nothing is ever sent externally. Every run is a dry-run that writes drafts and approval queues to local files. No income is guaranteed; all prices in linked documents are market guidelines.

The kit's job ends at drafts and checklists — an external send capability does not exist, by design. Reviewing, approving, and sending are always human steps, so a beginner cannot accidentally message a client.

One-command demo workspace

New simplest route: download the project and double-click START_HERE.command on macOS or START_HERE_WINDOWS.bat on Windows. You can also give START_WITH_AI.txt to Codex. The First Project guide explains all three routes.

ai-automation-kit complete-workspace --flow-id invoice-document-followup \
  --client-type local-business --niche accounting --output .tmp/complete-accounting
final_delivery_guide=.tmp/complete-accounting/FINAL_DELIVERY_GUIDE.md
status=ready_to_share

Flows: install → validate → run → approve

ai-automation-kit flows list --industry finance
ai-automation-kit flows diagram invoice-document-followup --output .tmp/diagram   # client-friendly diagram
ai-automation-kit flows install invoice-document-followup --output .tmp/first-job
ai-automation-kit flows validate .tmp/first-job
status=ready
ai-automation-kit flows run .tmp/first-job
automation_status=succeeded
draft_outputs=.tmp/first-job/automation_output/draft_outputs.md
approval_queue=.tmp/first-job/automation_output/approval_queue.csv
# read draft_outputs.md with your own eyes, then:
ai-automation-kit flows approve .tmp/first-job --approver you@example.com
approval_status=approved
outbox=.tmp/first-job/local_outbox/email_drafts.md
Only humans send. flows run supports dry-run only, and approve just records the review and fills a local outbox. Copy approved drafts into the real tool yourself.

Sales assets

ai-automation-kit beginner-sales --flow-id invoice-document-followup \
  --client-type local-business --niche accounting --output .tmp/beginner-sales
ai-automation-kit offer-pack --business-area operations --client-type small-business \
  --source-output .tmp/complete-accounting --output .tmp/offer-pack
ai-automation-kit business-launch --industry finance --client-type local-business \
  --niche accounting --output .tmp/business-launch
ai-automation-kit report-automation --report-type monthly \
  --client-type local-business --niche construction --output .tmp/monthly-report-pack

Generates discovery question sheets, one-page proposals, price menus, SOW, and risk boundaries. Prices in templates are Japanese market guidelines, not guarantees.

report-automation creates a folder-based report drafting pack with past completed reports, current materials, GrillMe-style questions, drafts, and human approval. See Report Automation Guide.

For the resumable local review loop, use ai-automation-kit report-wizard serve --workspace .tmp/report-wizard-weekly --language en --no-open and follow the Report wizard manual.
