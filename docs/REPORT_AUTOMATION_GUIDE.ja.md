# 日報・月報AI下書き自動化ガイド

このガイドは、AIを使い始めた人が「日報・週報・月報の作成支援」を副業として提案するための説明書です。

狙いは、完全自動提出ではありません。

狙いは、次の形です。

```text
過去の完成物を保存する
今回の資料を保存する
AIが過去の書き方を参考にして下書きする
AIが判断できない部分は GrillMe 方式で1問ずつ質問する
人間が確認して提出する
```

## なぜ副業で売りやすいのか

日報、週報、月報、業務報告は、多くの会社で毎日または毎月発生します。しかも、作業は地味ですが時間がかかります。

AI初心者がいきなり大きな自動化システムを売るのは難しいです。けれども「過去の月報と今月の資料をフォルダに入れたら、AIが下書きを作り、不明点だけ質問してくれる」という提案なら、相手にも伝わりやすいです。

## 使うコマンド

```bash
ai-automation-kit report-automation \
  --report-type monthly \
  --client-type local-business \
  --niche construction \
  --output .tmp/monthly-report-pack
```

作成されたフォルダで、まず次を開きます。

```text
.tmp/monthly-report-pack/START_HERE_REPORT_AUTOMATION.md
.tmp/monthly-report-pack/demo_report_automation.html
.tmp/monthly-report-pack/ai_agent_prompt.md
```

## フォルダの意味

| フォルダ | 入れるもの |
|---|---|
| `01_past_outputs/` | 過去に提出済みの日報、週報、月報 |
| `02_current_materials/` | 今回のCSV、議事録、写真、添付資料、作業ログ |
| `03_templates/` | 日報・月報の型 |
| `04_ai_analysis/` | AIが見つけた書き方、足りない情報、矛盾 |
| `05_grill_me_questions/` | AIが1問ずつ確認する質問 |
| `06_drafts/` | AIが作る下書き |
| `07_approval/` | 人間の承認チェック |

## GrillMe方式とは

GrillMe方式とは、AIが曖昧なまま進めず、1問ずつ人間に確認する進め方です。

悪い例:

```text
AIが売上減少の理由を勝手に「季節要因」と書く。
```

良い例:

```text
AIが止まって質問する。
「今月の売上が前月より下がった主な理由は何ですか？」
```

この仕組みにすると、AI初心者でも安全に企業へ提案しやすくなります。

## 売り込みやすい業種

- 建設業の日報、工事進捗報告
- 店舗の日報、売上報告
- ホテル、観光業の運営月報
- 営業チームの週報
- 製造業の作業日報
- 介護、福祉の記録整理
- SNS運用レポート
- 問い合わせ対応レポート

## 提案トーク

```text
過去の月報と今月の資料をフォルダに入れるだけで、AIが月報の下書きを作る仕組みを作れます。
AIが判断できない部分は勝手に書かず、1問ずつ確認します。
最後は人間が承認して提出するので、完全自動より安全に始められます。
まずは本番提出しない dry-run PoC から始めましょう。
```

## 売らない方がよい言い方

- AIが完全自動で月報を提出します
- AIが理由も数字も自動判断します
- 人間確認なしで提出できます
- 売上改善を保証します

このプロジェクトでは、AIは下書きと質問を担当し、最終判断は人間が行います。
