# 副業ブループリントガイド

このガイドは `side-hustle-blueprints` コマンドの説明です。

「業務自動化を副業として売りたいが、何を売ればよいか分からない」という人が、最初の案件候補を選ぶために使います。

## 生成コマンド

```bash
ai-automation-kit side-hustle-blueprints --industry local-business --operator-level beginner --output .tmp/side-hustle-blueprints
```

重要な生成ファイル:

- `START_HERE_SIDE_HUSTLE_BLUEPRINTS.md`: 最初に読むファイル
- `side_hustle_blueprints.md`: 売りやすい自動化案件カタログ
- `side_hustle_blueprints.html`: ブラウザーで見られる案件カタログ
- `side_hustle_blueprints.csv`: 表計算で見られる案件カタログ
- `first_client_picker.md`: 最初の顧客を選ぶための採点表
- `client_intake_questions.md`: 提案前に聞く質問リスト
- `implementation_paths.md`: ローカル、表計算、ワークフロー基盤、AIエージェント、クラウドの進め方
- `offer_scripts.md`: 初回営業やヒアリングで使う文章
- `risk_boundaries.md`: 人間が承認すべき範囲
- `ai_agent_handoff.md`: Codex、Claude Code、Cursor、ChatGPT、Claude などに渡す指示

## 何に役立つか

初心者がいきなり「何でも自動化できます」と売るのは危険です。

このコマンドは、広すぎる自動化アイデアを、売り込みやすい小さな案件に分けます。

例:

- ホームページと問い合わせ運用
- AI受付と一次返信
- 請求書・書類の追跡
- 社内FAQ
- 営業リサーチ
- 口コミ返信
- 予約リマインド
- 見込み客の振り分け
- レポートのダッシュボード化
- 書類受付とファイル整理
- 在庫と発注アラート
- 顧客オンボーディング

各案件には、売り込み先、困りごと、必要な入力、納品物、使う道具、価格帯、人間の承認範囲、最初の営業文が入ります。

## 初心者向けの考え方

「完全自動で会社の仕事を置き換えます」と売らないでください。

最初は小さく売ります。

1. 承認済みのサンプルデータを集める
2. 今の業務フローを書く
3. ローカル実行または表計算中心で小さく試す
4. 人間が出力を確認する
5. 時間短縮や抜け漏れ削減を確認する
6. その後で本番化やクラウド化を考える

## AIエージェントに渡す使い方

生成された `ai_agent_handoff.md` を AIエージェントに渡してください。

使いやすい依頼文:

> このGitHubプロジェクトと `side-hustle-blueprints` フォルダを確認して、初心者でも売りやすい自動化案件を1つ選ぶのを手伝ってください。質問は1つずつしてください。秘密情報や顧客の個人情報は聞かないでください。

## 公開情報の使い方

このブループリントは、公開されているワークフロー自動化、AIアプリ、AIエージェント、ブラウザ自動化、ローカル実行の考え方を参考にしています。公開プロジェクトは仕組みを学ぶための参考です。ブランド、非公開データ、独自業務をコピーするためのものではありません。
